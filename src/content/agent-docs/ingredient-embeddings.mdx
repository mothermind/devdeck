  - id: hybrid-search
    invocation: "GET /metrics/ingredients/hybrid-search?query=<string>"
    example: "GET /metrics/ingredients/hybrid-search?query=나이아신아마이드"

mechanisms:
  - id: model-bootstrap
    claim: On first instantiation IngredientMatcher checks app/data/models/klue-bert-base/config.json and app/data/embeddings/ingredient_embeddings.json; S3 download is the fallback only if local cache is missing.
    evidence: app/services/filtering.py:84-86,129-130
    rationale: Avoids S3 network calls on warm restarts; S3 remains the source of truth for the canonical embedding snapshot.
  - id: mean-pooling
    claim: Token embeddings are weighted by attention_mask, summed and divided by clamped mask sum, then L2-normalized via F.normalize(p=2, dim=1).
    evidence: app/services/filtering.py:151-155,167-168
  - id: cosine-search-with-boost
    claim: Similarity search is a full linear scan using torch.cosine_similarity with a default threshold of 0.9; an exact substring match applies a +0.2 boost capped at 1.0; results are returned sorted descending.
    invariant: Ingredients whose name literally contains the query string always rank above pure-semantic matches at the same cosine score.
    evidence: app/services/filtering.py:315-346
    rationale: Pure cosine similarity can under-rank ingredients whose name literally contains the query string; the boost ensures prefix/exact matches surface first.
  - id: ingestion-refresh
    claim: compute_and_save_embeddings fetches DISTINCT matched_answer from product_analysis WHERE question_id='CM00aa7ace476c60', processes them in batch_size=32 with max_length=64 tokenization, writes a timestamped JSON archive and canonical ingredient_embeddings.json locally, uploads both to S3, and updates the in-memory singleton dict.
    evidence: app/services/filtering.py:176-313
  - id: flat-json-storage
    claim: Embeddings are stored as a plain JSON dict (ingredient name -> float[]) on disk and in memory, not in a vector database or pgvector column. Similarity search is O(n) over the full dict.
    invariant: No ANN index or vector extension required in PostgreSQL; the entire embedding store is portable as a single JSON file.
    evidence: app/services/filtering.py:41-45,131-133,212,252
    rationale: Simplifies deployment, avoids a dedicated vector store dependency, and enables S3-based versioning with timestamped archives. Acceptable at the current corpus size (~6,256 ingredients).
  - id: singleton
    claim: get_ingredient_matcher() returns a module-level _ingredient_matcher_instance; IngredientMatcher is instantiated once per process on first request, not at startup.
    invariant: Model loading overhead (~seconds) is paid once per process; a slow/unreachable S3 at startup does not block FastAPI from serving other routes.
    evidence: app/services/filtering.py:350-357
    rationale: Lazy initialization avoids blocking FastAPI startup if S3 is unreachable at process start.
