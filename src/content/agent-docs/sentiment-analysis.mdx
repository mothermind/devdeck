mechanisms:
  - id: zero-shot-nli-scoring
    claim: Reviews are scored by evaluating NLI entailment probability against Korean-language hypothesis strings using MoritzLaurer/mDeBERTa-v3-base-xnli-multilingual-nli-2mil7; no fine-tuning or labeled training data is required.
    rationale: Enables classification against arbitrary Korean hypothesis strings without labeled training data; the multilingual model handles Korean natively. Entailment probability is used directly as the score for each label.
    evidence: utils/sentiment_analysis_gpu.py:22-68
  - id: fp16-gpu-inference
    claim: The model is converted to fp16 via model.half() on CUDA, with TF32 matmul, cuDNN benchmark, and AMP autocast enabled for maximum GPU throughput.
    rationale: Reduces GPU memory footprint and increases throughput on Ampere hardware; combined with TF32 matmul and cuDNN benchmark flags for maximum CUDA utilization.
    evidence: utils/sentiment_analysis_gpu.py:30-34, utils/sentiment_analysis_gpu.py:53-55
  - id: dynamic-entailment-index
    claim: The entailment label index is resolved at runtime from model.config.id2label rather than hardcoded, ensuring the correct probability column is extracted regardless of label ordering.
    invariant: Correct entailment probability is always extracted even if the model checkpoint reorders NLI output labels.
    rationale: Avoids hardcoding the NLI label order; a prior hardcoded version (probs[:, 2]) was replaced by this approach.
    evidence: utils/sentiment_analysis_gpu.py:48-49
  - id: async-semaphore-batching
    claim: Inference is dispatched as concurrent async batch tasks via asyncio, with in-flight GPU work bounded by asyncio.Semaphore(max_concurrent=200) and outer memory bounded by chunk_size=50000 rows.
    invariant: GPU memory usage is bounded regardless of dataset size; no more than 200 batch tasks are in-flight concurrently.
    rationale: Allows large datasets to be queued as concurrent batch tasks while bounding maximum in-flight GPU work via semaphore; outer chunking keeps memory bounded.
    evidence: "utils/sentiment_analysis_gpu.py:59-61, utils/sentiment_analysis_gpu.py:110-142"
  - id: keyword-column-passthrough
    claim: Keyword columns produced by the upstream preprocessing script are forwarded unchanged into the sentiment output DataFrame so the downstream PostgreSQL table receives a wide schema combining both keyword and sentiment signals.
    rationale: Downstream PostgreSQL table expects a wide schema combining both keyword and sentiment signals; the sentiment script preserves upstream keyword columns by copying them into results_df.
    evidence: "utils/sentiment_analysis_gpu.py:283-287, main_gpu.py:187-191"
