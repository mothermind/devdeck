mechanisms:
  - id: category-level-progress
    claim: Progress is checkpointed at category granularity (not per-product); per-product deduplication is re-run against DynamoDB and S3 on every resume.
    invariant: No extra per-product state file is needed; safe reruns are ensured by re-checking DynamoDB and S3 existence on each product.
    evidence: "main.py:72-96, main.py:827-842"
    rationale: Avoids persisting a per-product checkpoint file; instead deduplication against DynamoDB is re-run per product on resume so no extra state file is needed.
  - id: triple-existence-dedup
    claim: Before any network work, the scraper checks DynamoDB AND both S3 keys (main + detail) and skips the product entirely if all three exist.
    invariant: Safe reruns mid-catalog without re-uploading images or re-inserting DB records.
    evidence: main.py:421-429
    rationale: Allows safe reruns mid-catalog without re-uploading images or re-inserting DB records.
  - id: shared-browser-context
    claim: A single Playwright browser context is shared across all category crawls; per-product pages are opened and closed inside the scrape_product wrapper.
    invariant: Browser launch overhead is paid once; per-page isolation is preserved for parallel tasks.
    evidence: "main.py:756-818, main.py:527-546"
    rationale: Reduces browser launch overhead while keeping per-page isolation for parallel tasks.
  - id: detail-image-stitch
    claim: Detail images are scrolled from #tab-panels, downloaded individually via aiohttp, then stitched vertically into a single JPEG before S3 upload.
    invariant: One composite image per product replaces many individual detail images in S3.
    evidence: "main.py:274-306, main.py:449-462, utils/utility_functions.py:178-211"
    rationale: Single composite image is cheaper to store and retrieve than many individual detail images.
  - id: anti-detection-stealth
    claim: Browser is launched with --disable-blink-features=AutomationControlled and stealth JS overrides for navigator.webdriver, navigator.plugins, navigator.languages, and window.chrome; user-agent spoofed as Chrome 120 on Windows 10; locale ko-KR; timezone Asia/Seoul.
    evidence: main.py:756-815
    rationale: Reduces detection probability on the Olive Young site.
  - id: captcha-pause
    claim: CAPTCHA handling detects 6 selector patterns and pauses 30s for manual intervention if any are found.
    evidence: main.py:485-509
  - id: image-retry-backoff
    claim: aiohttp image downloads retry up to 3 times with exponential backoff (2^attempt seconds sleep) on failure.
    evidence: main.py:349-360
  - id: s3-key-pattern
    claim: "S3 key pattern for main product images is images/olive_young/product_main/{category_id[:-4]}/{category_id[-4:]}/{item_id}.jpg"
    evidence: main.py:418-419
  - id: hardcoded-dynamodb-table
    claim: DynamoDB table name 'OliveYoung_MetaData' is hardcoded in OliveYoungScraper.__init__ and cannot be overridden via CLI or env var.
    evidence: "main.py:144, utils/utility_functions.py:73-87"
    rationale: No apparent rationale — likely an expedient choice.
